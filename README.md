Интеллектуальная система модерации и аналитики коммуникаций (ИИ + Человек в контуре управления)


Проект представляет собой интеллектуальную систему, объединяющую аудиторию, маркетинг, продуктовую команду и отдел заботы о пользователях. Это стратегическое решение направлено на комплексную оптимизацию работы маркетинга, повышение лояльности клиентов и их долгосрочное удержание. 

Система автоматически классифицирует и обрабатывает входящий поток комментариев в социальных сетях компании (продажа билетов на мероприятия), создавая единый кросскомандный фокус на реальных потребностях пользователя.



Цель проекта
Разработать систему на основе искусственного интеллекта для автоматической обработки входящих обращений пользователей в социальных сетях компании. 

Система должна в реальном времени определять потребности (интенты) клиентов и формировать точные ответы, решая комплексную бизнес-задачу: кратно снизить операционную нагрузку на SMM-специалиста, защитить маркетинговые бюджеты от потери лидов и поставлять непрерывные продуктовые инсайты для команды разработки.



Задачи
Проектирование ML-ядра
Разработка и внедрение архитектуры, разделяющей задачи фильтрации и маршрутизации:
Шаг 1: Бинарный классификатор для определения информационного шума (отсечение спама, флуда и нерелевантных комментариев).
Шаг 2: Мультиклассовая маршрутизация обращений по 9 ключевым бизнес-сценариям компании (Возврат билетов, Проблема со входом в приложение, Оплата баллами и т.д.).

2. Внедрение системы контроля качества моделей
Настройка автоматического контура тестирования нейронной сети для беспрерывного мониторинга точности классификации.

3. Разработка внутреннего пространства для маркетолога
Создание защищенного рабочего места оператора с логикой перехвата сообщений из публичных Telegram-каналов и их маршрутизацией в приватный интерфейс администратора.

4. Внедрение механики «Человек в контуре» (Human-in-the-Loop)
Обеспечение 100% гарантии качества (SLA) клиентского сервиса. Нейросеть не отвечает пользователю напрямую, а формирует «карточку обращения» с предсказанным интентом и шаблоном из базы эталонных ответов. Маркетолог имеет возможность:
Подтвердить автоответ в один клик (максимальная скорость).
Скорректировать бизнес-интент или отредактировать финальный текст.
*Данный архитектурный паттерн полностью исключает репутационные риски, связанные с «галлюцинациями» ИИ. Нейросеть берет на себя маршрутизацию, а человек сохраняет финальный контроль. Таким образом, мы минимизируем риски выйти в инфо-поле с некорректным ответом.

5. Построение хранилища данных и аналитики
Интеграция базы данных для автоматического логирования всех метрик сессии (текст, флаг спама, уверенность модели, ручные вмешательства оператора) и генерация управленческих отчетов по запросу внутри Telegram-бота. 

Функциональность
Архитектура продукта состоит из четырех взаимосвязанных модулей, каждый из которых работает на снижение издержек, рост вовлеченности (ER) и передачу важных продуктовых инсайтов между отделами:

1. ML-ядро: Оптимизация воронки и защита лидов (Двухконтурный конвейер)
Уровень 1:Бинарная детекция информационного шума. Нейросеть автоматически распознает флуд и нерелевантные комментарии, маркируя их как мусор. Это освобождает SMM-отдел от рутинного разбора спама и позволяет сфокусировать 100% внимания на реальных запросах клиентов.
Уровень 2: Точная маршрутизация полезных обращений по 9 ключевым сценариям: *Возврат билета, Возврат подаренного билета, Оплата бонусными баллами, Подарок билета, Объявление в знакомствах, Проблема с билетом, Проблема со входом в приложение, Проблема с проходом на мероприятие, Связь с поддержкой.* Моментальная классификация удерживает "горячих" пользователей и защищает рекламный бюджет (снижение CAC).

2. Контур контроля качества: Защита клиентского опыта (MLOps & QA)
Встроенный пайплайн автоматического тестирования модели гарантирует бесперебойную точность алгоритмов, предотвращая ситуации, когда клиент получает неверный ответ. Стабильность системы напрямую работает на метрики удержания (Retention Rate) и удовлетворенности (CSAT).

3. Рабочее место оператора: Интерфейс удержания клиентов
Автоматический перехват сообщений из соцсетей и их безопасная маршрутизация в приватный Telegram-чат администратора.
Формирование «умной карточки»:** система предсказывает интент и автоматически подтягивает готовый шаблон эталонного ответа.
Оператор может в один клик отправить ответ, вручную скорректировать категорию или отредактировать текст. Это радикально снижает время первого ответа, напрямую повышая лояльность и NPS.

4. Модуль сквозной аналитики: Межкомандный фокус 
Бесшовное фоновое логирование всех параметров сессии (тексты, флаги спама, уверенность модели, ручные правки) в базу данных SQLite.
Генерация дашбордов (Executive Summary):** Вывод метрик автоматизации и статуса очереди прямо в Telegram.Синергия отделов:** Накопленные данные о проблемах (например, всплеск интента "Проблема со входом") автоматически становятся ценными инсайтами:
  Для IT/Продукта:* быстрый поиск UX-багов в приложении.
  Для Маркетинга:* сигнал к созданию обучающих гайдов или триггерных рассылок.
  Для ML-отдела:* чистый датасет для непрерывного дообучения алгоритма (Active Learning).



Архитектура и Технологический стек

Machine Learning & NLP:** `PyTorch`, `Hugging Face Transformers`, `scikit-learn`.
Модель:Легковесная `cointegrated/rubert-tiny2` (обеспечивает высочайшую скорость инференса на CPU).
Backend & Бэкофис: `Python 3.10+`, `aiogram 3.x`, `asyncio` (асинхронная обработка пиковых нагрузок).
Data Storage & Analytics: `SQLite3`, `Pandas` (надежное in-file хранилище).
ML QA: `unittest`, `mock`.

Структура проекта разделена на ML-ядро и интерфейс Telegram-бота.

1. Установите зависимости:
   ```bash
   pip install -r requirements.txt


Убедитесь, что файл с эталонными ответами Answers_hackaton_final.csv находится в корневой директории со скриптами.

Прогоните авто-тесты, обучите модель и инициализируйте пайплайн:

python ml.py

(После успешного завершения будут сохранены веса модели и логи тестирования).

Запустите бэкофис модератора (Telegram-бота):
python tg_bot.py

Напишите вашему боту сообщение от лица пользователя. Для выгрузки бизнес-аналитики по сессиям отправьте боту команду /db.
